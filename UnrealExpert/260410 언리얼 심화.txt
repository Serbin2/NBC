리플렉션이란? : 런타임에 프로그램이 자신의 구조를 인지하고 조작할 수 있게 하는 능력
주요 역할 : 런타임 타입 정보 확인 / 에디터 연동/ 가비지 컬렉션 지원

가비지컬렉션
BeginDestroy() 삭제 대기 상태가 된다
FinisheDestroy() GC가 삭제 완료

TObjectPtr : GC가 감시하고 참조를 증가시켜 삭제되지 않도록 한다
TWeakObjectPtr : 감시는 하되 참조를 증가시키지 않아 삭제가 될 수 있다
WeakPtr 사용 예시 : 적 유닛을 저장하여 사용 할 때, 적 유닛은 죽어 사라질 수 있으므로 -> 이런 상황에 위크포인터 사용

UBT(언리얼 빌더 툴)
가장 먼저 실행되는 빌드 도구
전체 프로젝트의 구조 파악
플랫폼별 환경 설정

주요 기능 : 
플러그인 및 모듈 검색
.Target.cs 분석
등

UHT(언리얼 헤더 툴)와 메타데이터
코드를 에디터가 이해할 수 있게 번역
UHT의 역할:
메타 데이터 수집
.generated.h 파일 자동 생성
리플렉션


빌드 타임라인과 라이브 코딩
1. UBT실행 -> 2. UHT 처리 -> 3. Visual Studio 컴파일 -> 4. 리플렉션 완성

헤더파일 변경 시 -> 전체 리빌드 (느림)
CPP 파일만 변경 시 -> 라이브 코딩 ( 2번과 3번 과정을 bypass, 빠름)


CDO (Class Default Object )
UObject 클래스마다 하나씩 존재하는 기본값 저장소
모듈 로딩 단계에서 생성
엔진이 켜지는 시점에 생성자 딱 한번만 호출됨

CDO를 활용한 최적화 원리
메모리 절약   /   빠른 초기화  /  델타 직렬화

```c++
// 예제 코드
AMyTestActor* ActorA = GetWorld()->SpawnActor<AMyTestActor>();
	UE_LOG(LogTemp, Warning, TEXT("ActorA Init Health : %d"), ActorA->Health);
	
	ActorA->Health = 50;
	UE_LOG(LogTemp, Warning, TEXT("ActorA Health : %d"), ActorA->Health);
	
	AMyTestActor* MyActorCDO = GetMutableDefault<AMyTestActor>();
	UE_LOG(LogTemp, Warning, TEXT("ActorCDO Health : %d"), MyActorCDO->Health);
	
	MyActorCDO->Health = 200;
	UE_LOG(LogTemp, Warning, TEXT("ActorCDO Health : %d"), MyActorCDO->Health);
	
	AMyTestActor* ActorB = GetWorld()->SpawnActor<AMyTestActor>();
	UE_LOG(LogTemp, Warning, TEXT("ActorB Health : %d"), ActorB->Health);
	UE_LOG(LogTemp, Warning, TEXT("ActorA Current Health : %d"), ActorA->Health);
```
